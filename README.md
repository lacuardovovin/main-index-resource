# main-index-resource

## 项目简介

`main-index-resource` 是一个用于归档和发布多个独立 HTML 页面的仓库。它不针对特定域名或网站，仅作为静态资源的集合点。每个 HTML 页面可以包含独立的内容、演示或工具，适合用于测试、展示或历史记录存档。

## 目录说明

```
.
├── index.html          # 仓库首页（可选，用于导航）
├── pages/              # 存放独立 HTML 页面的目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 页面所需的资源文件（CSS、JS、图片等）
│   ├── styles/
│   ├── scripts/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立的 HTML 页面存放于此，文件名应保持唯一且具描述性。
- `assets/`：页面引用的样式、脚本、图片等资源，按需组织子目录。

## 页面归档说明

本仓库中的每个 HTML 页面均为独立单元，可单独通过路径访问（例如：`https://<你的托管地址>/pages/example-1.html`）。页面之间无依赖关系，内容由贡献者自行决定。归档时请遵守以下原则：

- 页面内容应合法、合规，不包含恶意代码或侵犯他人权益的信息。
- 页面资源应尽量内联或存放在 `assets/` 目录下，避免引用外部不稳定资源。
- 每个页面建议在文件头部添加注释说明其用途和创建日期。

## 维护说明

- **添加页面**：在 `pages/` 下新建 HTML 文件，如有资源请放入 `assets/` 对应子目录，并更新仓库首页（如存在）。
- **更新页面**：直接修改对应文件，注意保持向后兼容或记录版本变更。
- **删除页面**：移除对应 HTML 文件及关联资源，避免残留引用。
- **提交规范**：提交信息应简洁描述变更内容，例如“新增页面：xxx 演示”或“更新页面：修复样式问题”。
